Installation

Requirements

MyProject requires Node.js ≥ 18 and a modern package manager. It runs in browser and server environments.

npm / yarn / pnpm

npm install myproject        # npm
yarn add myproject           # yarn
pnpm add myproject           # pnpm

For a CDN-based setup (no bundler required):

<script type="module">
  import { create } from 'https://cdn.jsdelivr.net/npm/myproject/+esm'
</script>

Configuration

Create a myproject.config.js in your project root:

export default {
  theme: 'warm',        // 'warm' | 'cool' | 'neutral'
  plugins: [],          // array of plugin instances
  debug: false,         // enable verbose logging
  output: './dist',     // build output directory
}
Note: All config keys are optional — MyProject ships with sensible defaults for every option.

Basic Usage

Creating an instance

Import create and pass your config. The returned instance exposes the full API.

import { create } from 'myproject'

const app = create({ theme: 'warm' })

// Register a handler
app.on('ready', () => console.log('App is ready'))

// Start the app
await app.run()

Working with data

Use the store module for reactive state:

import { store } from 'myproject'

const state = store({ count: 0 })

state.subscribe(s => console.log('count:', s.count))
state.set({ count: state.get().count + 1 })

Advanced Patterns

Plugin authoring

Plugins are plain objects with a name and setup function. The setup function receives the app instance and can hook into any lifecycle event.

const myPlugin = {
  name: 'my-plugin',
  setup(app) {
    app.on('ready', () => {
      // plugin logic here
    })
  }
}

const app = create({ plugins: [myPlugin] })

Code splitting

MyProject supports lazy module loading out of the box. Use app.lazy() to defer expensive imports:

const heavyModule = await app.lazy(() => import('./heavy'))

API Reference

  • create(config?) — Creates and returns a new app instance.
  • app.run() — Starts the application. Returns a Promise.
  • app.on(event, handler) — Registers a lifecycle event handler.
  • app.off(event, handler) — Removes a lifecycle event handler.
  • app.lazy(loader) — Lazily imports a module.
  • store(initial) — Creates a reactive store with get, set, subscribe.

FAQ

Does it work with React / Vue?

Yes. MyProject is framework-agnostic. Official adapters exist for React, Vue 3, and Svelte — see the Projects page.

Is it production ready?

Absolutely. MyProject is used in production by dozens of teams. The core API has been stable since v2.0.
